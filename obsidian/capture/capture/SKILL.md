# Capture — 想法记录 + 发散

## 执行流程

### Step 1: 提取想法内容

从用户消息中提取想法文本。若没有内容，询问：
> 你想记录什么想法？

### Step 2: 运行脚本

```bash
export PATH="$HOME/.bun/bin:$PATH"
bun run "${SKILL_DIR}/scripts/capture.ts" "<想法内容>"
```

脚本会：
1. 将想法追加写入 `00_收集箱/想法.md`，格式如下：
   ```
   ## YYYY-MM-DD HH:mm
   <想法内容>
   #待发散
   ```
2. 检索 `20_素材库/` 和 `10_工具库/已验证/` 中的相关内容
3. 输出 `RELATED:[...]` 供后续使用

### Step 3: 输出结果

解析脚本输出中的 `RELATED:[...]`，然后用中文输出：

```
已记录到收集箱 ✓

[关联] （如果 RELATED 不为空，列出相关素材/工具；否则跳过这一行）

[发散] 三个方向：
1. （贴近直觉：顺着想法最自然的延伸）
2. （反直觉：对立视角或挑战假设）
3. （跨场景迁移：把这个想法用在另一个完全不同的场景）

要展开某个方向吗？
```

### Step 4: 展开（如果用户选择）

用户说"展开1"/"展开2"/"展开3"或直接描述想展开哪个方向：

- 继续对该方向发散，提出 2-3 个更具体的问题或可能性
- 保持发散者角色，不急于收敛
- 对话自然结束时询问：

> 要把这次讨论的要点追加到收集箱吗？

若用户同意，在 `00_收集箱/想法.md` 对应想法下追加：

```
### 头脑风暴
- <要点1>
- <要点2>
- <关键问题>：<问题内容>
```

## 注意事项

- 发散的三个方向必须来自**不同维度**，不是同一方向的三种说法
- 不要在发散阶段就建议"创建项目"或"写文章"，保持探索状态
- 关联内容有则显示，无则跳过，不强行关联
